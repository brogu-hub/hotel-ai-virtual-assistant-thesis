# pydantic_models — Request/Response Schemas

`src/hotel_guardrails/models.py` defines every Pydantic model used by the hotel_guardrails FastAPI server. The module covers 30+ schemas spanning chat, booking, rooms, guests, sessions, authentication, admin operations, and dashboard statistics. All models inherit from `pydantic.BaseModel` and use `Field(...)` descriptors with descriptions and examples suited for OpenAPI documentation.

---

## Core Chat Models

### `LLMOverrides`

Optional per-request LLM configuration. Nested inside `ChatRequest.llm_settings`.

| Field | Type | Constraints | Purpose |
|---|---|---|---|
| `model` | `Optional[str]` | — | Override model for this request (e.g. `"qwen/qwen3-max"`) |
| `temperature` | `Optional[float]` | `ge=0.0`, `le=2.0` | Sampling temperature |
| `max_tokens` | `Optional[int]` | `ge=1`, `le=8192` | Maximum output tokens |

### `ChatRequest`

Request body for `POST /chat` and `POST /chat/stream`.

| Field | Type | Constraints | Purpose |
|---|---|---|---|
| `message` | `str` | `min_length=1`, `max_length=4096` | User message (Thai or English) |
| `session_id` | `Optional[str]` | — | Conversation continuity ID; auto-generated by server if omitted |
| `user_id` | `Optional[str]` | `max_length=128` | **Stable guest identifier for cross-session long-term memory** |
| `language` | `Optional[str]` | `pattern="^(th\|en\|auto)$"`, default `"auto"` | Preferred response language |
| `llm_settings` | `Optional[LLMOverrides]` | — | Per-request model/temperature/token overrides |

> [!note]
> `user_id` is the memory-system key. When present it is stored under the `("guest", user_id)` namespace in the LangGraph store, which has **no expiry**. When absent the server substitutes `"guest"` at the call site, creating an anonymous namespace entry `("anon", session_id)` eligible for 30-day TTL cleanup by the scheduled sweeper. See [[guest_memory_store]] for the full namespace design.

### `ChatResponse`

Response body for `POST /chat`.

| Field | Type | Purpose |
|---|---|---|
| `response` | `str` | AI-generated text in Thai or English |
| `session_id` | `str` | Session ID for subsequent requests |
| `sources` | `Optional[List[str]]` | RAG source document names |
| `retrieval_context` | `Optional[List[str]]` | Raw retrieved chunks (for DeepEval evaluation) |
| `tool_calls` | `Optional[List[Dict[str, Any]]]` | Tool invocations made during the turn |
| `request_id` | `Optional[str]` | Echoes `X-Request-ID` header for tracing |
| `routing_path` | `Optional[str]` | `"langgraph"`, `"langchain_fallback"`, `"admin_override"`, or `"blocked"` |
| `routing_reason` | `Optional[str]` | Human-readable routing decision explanation |
| `complexity` | `Optional[str]` | `"simple"`, `"moderate"`, or `"complex"` |

### `StreamChunk`

SSE payload for `POST /chat/stream`. Each event is a JSON-serialised `StreamChunk`.

| Field | Type | Purpose |
|---|---|---|
| `content` | `str` | Partial response text |
| `done` | `bool` | `True` on final chunk |
| `session_id` | `Optional[str]` | Present only on the final chunk |

## Booking Models

### `BookingRequest`

Request for `POST /tools/book`. The `action` field gates which other fields are required.

| Field | Type | Constraints | Required for |
|---|---|---|---|
| `action` | `str` | `pattern="^(check\|create\|confirm\|cancel)$"` | All |
| `guest_id` | `Optional[str]` | — | `create` |
| `room_type` | `Optional[str]` | — | `check` |
| `room_id` | `Optional[str]` | — | `create` |
| `reservation_id` | `Optional[str]` | — | `confirm`, `cancel` |
| `check_in` | `Optional[date]` | — | `check`, `create` |
| `check_out` | `Optional[date]` | — | `check`, `create` |
| `special_requests` | `Optional[str]` | `max_length=1000` | Optional |
| `reason` | `Optional[str]` | `max_length=500` | Optional for `cancel` |

### `BookingResponse`

Response for `POST /tools/book`.

| Field | Type | Purpose |
|---|---|---|
| `success` | `bool` | Operation outcome |
| `action` | `str` | Echo of the requested action |
| `data` | `Optional[Dict[str, Any]]` | Action-specific result payload |
| `message` | `str` | Human-readable bilingual message |
| `reservation_id` | `Optional[str]` | Set when action creates or affects a reservation |

### `BookingInfo`

Detailed reservation record returned from `GET /bookings/{reservation_id}`.

| Field | Type | Notes |
|---|---|---|
| `reservation_id` | `int` | DB primary key |
| `confirmation_number` | `str` | Human-readable code e.g. `HTL240215001` |
| `guest` | `GuestInfo` | Nested guest profile |
| `room_number`, `room_type` | `str` | Assigned room details |
| `check_in_date`, `check_out_date` | `str` | YYYY-MM-DD |
| `num_nights`, `num_guests` | `int` | Derived / stored values |
| `status` | `str` | `pending \| confirmed \| checked_in \| checked_out \| cancelled` |
| `total_amount` | `float` | THB |
| `payment_status` | `str` | `pending \| paid \| refunded` |
| `special_requests`, `cancellation_reason` | `Optional[str]` | Free text |
| `created_at`, `updated_at` | `str` | ISO timestamps |

### `BookingUpdateRequest`

`PATCH /bookings/{reservation_id}` — all fields optional, only changed fields required.

| Field | Type | Constraints |
|---|---|---|
| `check_in_date`, `check_out_date` | `Optional[date]` | — |
| `room_number` | `Optional[str]` | — |
| `num_guests` | `Optional[int]` | `ge=1`, `le=10` |
| `special_requests` | `Optional[str]` | `max_length=1000` |

### `BookingUpdateResponse`

| Field | Type | Purpose |
|---|---|---|
| `success` | `bool` | — |
| `message` | `str` | Bilingual confirmation |
| `booking` | `Optional[BookingInfo]` | Updated booking state |
| `changes` | `Dict[str, Any]` | Old/new value diff |

### `BookingListResponse`

Paginated list of `BookingInfo` objects with `total`, `page`, `page_size`.

## Room Models

### `RoomTypeResponse`

Room type catalogue entry.

| Field | Type | Purpose |
|---|---|---|
| `room_type_id` | `int` | DB identifier |
| `name`, `name_th` | `str`, `Optional[str]` | English and Thai names |
| `description`, `description_th` | `Optional[str]` | Bilingual descriptions |
| `base_price` | `float` | Per-night price in THB |
| `max_occupancy` | `int` | Guest cap |
| `size_sqm` | `Optional[int]` | Room floor area |
| `bed_type`, `view_type` | `Optional[str]` | Configuration details |
| `amenities` | `List[str]` | e.g. `["WiFi", "TV", "Minibar"]` |
| `photos` | `List[str]` | Photo URLs |
| `available_count` | `Optional[int]` | Current availability count |

### `RoomResponse`

Individual room instance including status and last-cleaned timestamp.

### `RoomDetailResponse`

Wraps `RoomResponse` with a `pricing` dict (base price, tax rate, service charge breakdown) and optional 30-day `availability` calendar.

### `RoomListResponse`

`GET /rooms` response: `rooms: List[RoomTypeResponse]`, `total: int`.

### `RoomAvailabilityResponse` / `RoomAvailabilityDay`

Calendar view for a date range. Each `RoomAvailabilityDay` carries `date`, `available`, `available_count`, `min_price`.

## Guest Models

### `GuestInfo`

Embedded in `BookingInfo`. Includes loyalty tier and points.

### `GuestResponse`

Full guest profile returned from guest endpoints. Includes nationality, Thai name fields, loyalty programme data.

### `GuestCreateRequest`

`POST /guests`: `email` (required, unique key), `first_name`, `last_name`.

### `GuestCreateResponse`

`success`, `message`, and nested `GuestResponse`.

### `GuestUpdateRequest`

`PATCH /guests/{guest_id}` — all fields optional: names (EN + TH), phone, nationality, address.

---

## Session Models

### `SessionCreateResponse`

`POST /sessions` — returns `session_id` (UUID) and `created_at` (ISO).

### `SessionInfo`

`GET /sessions/{id}` — adds `last_activity` and `message_count` to session metadata.

### `ConversationMessage`

Single message in a session: `role` (`user \| assistant`), `content`, `timestamp`, optional `metadata` (sources, tool_calls).

### `ConversationHistoryResponse`

`GET /sessions/{id}/messages` — `session_id`, `messages`, `total`, `has_more`.

---

## Authentication Models

### `UserRegisterRequest`

`POST /auth/register` — `username` (alphanumeric + `.`, `_`, `-`; 3–64 chars), `email`, `password` (8–128 chars), `full_name?`.

### `UserLoginRequest`

`POST /auth/login` — `username` (accepts email address), `password`.

### `UserResponse`

Public user record (no `password_hash`): `user_id`, `username`, `email`, `role` (`user \| admin`), `full_name`, `is_active`, `guest_id` (link to guest profile), `last_login`, `created_at`.

### `TokenResponse`

`POST /auth/login` and `/auth/register` response: `access_token` (JWT), `token_type="bearer"`, `expires_in` (seconds), `user: UserResponse`.

### `PasswordChangeRequest`

`PATCH /auth/me/password` — `current_password` (for verification), `new_password` (8–128 chars).

---

## LLM Settings Models

### `LLMSettingsResponse`

`GET /settings/llm` — `backend` (`"ollama"` or `"openrouter"`), `model`, `temperature`, `max_tokens`, `streaming`, `thinking` (reasoning mode flag), `ollama_base_url?`, `openrouter_model?`, `available_models`.

### `LLMSettingsUpdateRequest`

`PUT /settings/llm` — optional `backend`, `model`, `temperature`, `max_tokens`. Changes take effect on the `RuntimeLLMConfig` singleton without restart.

| `AdminBookingStatusRequest` | `PATCH /admin/bookings/{id}/status` | `status`, `notes?` |

## Dashboard Models

### `DashboardStatsResponse`

`GET /dashboard/stats` — hotel-wide snapshot: room status breakdown, reservation status breakdown, today's check-ins/checkouts/new bookings, total and today's revenue, total guests, service request stats, occupancy rate.

### `RecentBookingItem`

Lightweight booking entry for the dashboard feed: `reservation_id`, `confirmation_number`, `status`, dates, `total_amount`, guest name and email, room number/type.

### `SessionStatsResponse`

`GET /dashboard/sessions/stats` — `total_sessions`, `total_messages`, split by `user_messages`, `bot_messages`, `admin_messages`.

---

## Utility Models

### `HealthResponse`

`GET /health` — `status` (`healthy \| degraded \| unhealthy`), `components: Dict[str, str]`.

### `ErrorResponse`

Returned by global exception handlers — `error` (type key), `message`, `details?`.

### `GenericSuccessResponse`

Generic `{success: bool, message: str}` for operations without domain-specific return data (e.g. logout, password change).

### `RoomAmenity`

Nested amenity object with `name`, `name_th`, `icon`. Used internally; not currently emitted as a top-level response.

## Notes

> [!note]
> The OpenAPI YAML snapshot at `docs/api_references/hotel_guardrails_server.yaml` omits the `user_id` field from the `ChatRequest` schema (it was added after the snapshot was last exported). The live `/docs` endpoint reflects the current model including `user_id`. Always prefer the live spec or the source model over the static YAML for `ChatRequest`.

> [!note]
> Bilingual `name_th`, `description_th`, `first_name_th`, `last_name_th` fields exist throughout the models, reflecting the Thai/English bilingual nature of The Grand Horizon Hotel deployment. Thai fields are always `Optional[str]`.
